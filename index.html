Research Prototype

Macro Allocation Dashboard

A Python-based macro allocation dashboard designed to test whether interpretable signals can improve portfolio allocation across SPY, Gold, and a capped Crypto satellite. The focus is not black-box prediction, but auditable decision logic and robustness testing.

Open Dashboard View Methodology

Model Structure

The dashboard is divided into two allocation views. Dashboard 1 is the cleaner core model, allocating between SPY and Gold. Dashboard 2 adds Crypto as a capped satellite sleeve, rather than treating it as a full core allocation asset.

Dashboard 1
SPY + Gold core allocation
Dashboard 2
SPY + Gold + capped Crypto satellite
Purpose
Transparent macro allocation testing

Signal Framework

The model combines macro and market-based inputs, including liquidity, real-yield changes, USD momentum, Gold trend/reversal signals, and Crypto momentum conditions.

Liquidity Real Yields DXY / USD Momentum Gold Timing Crypto Momentum Regime Filters

Validation Layer

The dashboard includes several robustness checks to reduce the risk of overfitting and to make the model easier to audit.

Backtest Quality
Same-period benchmark comparison and non-overlapping 3M returns
Robustness
Train / validation / test split and walk-forward parameter validation
Explainability
Sensitivity table, regime breakdown, and signal contribution

Current Research Takeaway

The SPY + Gold strategy is the cleaner institutional-style core framework. Crypto can improve upside in certain periods, but it should remain capped because of drawdown and regime risk.
